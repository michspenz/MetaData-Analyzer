# Metadata Analyzer

> **Forensic metadata extraction tool for images and documents — Digital Forensics Portfolio Project**

---

## Project Overview

In digital forensics, **metadata** is data about data. Every file carries hidden information embedded at creation: who created it, when, with what device or software, and sometimes — exactly where on Earth. This information is invisible during normal use but invaluable during an investigation.

**Metadata Analyzer** is a Python-based forensic tool that surfaces this hidden layer from common file types and compiles it into structured, investigator-ready reports. Use cases include:

- **Incident Response** — determine when a file was created or last modified
- **Evidence Attribution** — identify document authors or camera make/model
- **OSINT / Geolocation** — extract GPS coordinates embedded in images
- **Chain of Custody** — timestamp and record file provenance
- **Malware Analysis** — identify compilation tool chains from document metadata

---

## Features

| Format | Extracted Fields |
|--------|-----------------|
| **JPG / JPEG / PNG / TIFF** | File name, size, resolution, format, creation/modification dates, camera make & model, lens, software, GPS coordinates (lat/lon/altitude), exposure, ISO, focal length, flash |
| **PDF** | Title, author, creator, producer, subject, keywords, creation/modification dates, page count, PDF version, encryption status |
| **DOCX** | Title, author, last modified by, description, keywords, category, creation/modification/print dates, revision number, word/paragraph/character count, application name |

**Report Formats:**
- **JSON** — structured per-file report
- **CSV** — flat summary across all analyzed files

## Installation

### Prerequisites

- Python 3.10 or later
- pip

### Steps

```bash
# 1. Clone the repository
git clone https://github.com/michspenz/metadata-analyzer.git
cd metadata-analyzer

# 2. (Optional but recommended) Create a virtual environment
python -m venv venv
source venv/bin/activate        # Linux / macOS
venv\Scripts\activate           # Windows

# 3. Install dependencies
pip install -r requirements.txt

# 4. (Optional) Generate sample test files
python generate_samples.py
```

## Usage

### Analyze a single file

```bash
python analyzer.py image.jpg
python analyzer.py document.pdf
python analyzer.py report.docx
```

### Analyze a folder (all supported files)

```bash
python analyzer.py sample_files/
```

Folder mode automatically generates a bulk JSON report and a CSV summary in `reports/`.

### Save an individual JSON report

```bash
python analyzer.py image.jpg --json
```

### Save a CSV summary for a single file

```bash
python analyzer.py image.jpg --csv
```

### Suppress terminal output (report-only mode)

```bash
python analyzer.py sample_files/ --no-print
```

### Full option reference

```
positional arguments:
  target        File or folder to analyze.

options:
  --json        Save JSON report(s) to the reports/ directory.
  --csv         Save a CSV summary to the reports/ directory.
  --no-print    Suppress pretty-print output to stdout.
  --help        Show help message and exit.
```

## Running in VS Code

1. Open the project folder in VS Code: `File → Open Folder`
2. Open the integrated terminal: `` Ctrl+` ``
3. Install dependencies: `pip install -r requirements.txt`
4. Run the tool:
   ```bash
   python analyzer.py sample_files/
   ```
5. Reports appear in `reports/` — visible in the Explorer panel on the left.

**Recommended VS Code Extensions:**
- Python (Microsoft)
- Pylance
- GitLens
